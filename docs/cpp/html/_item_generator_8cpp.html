Common template-based processing logic for LLM responses.

This template function extracts the common logic from ProcessLLMResponse_* functions. Type-specific operations are passed as function objects/lambdas. All item types now use qualityResults for unified invalid item filtering.

Template Parameters
ItemTypeThe item data type (ItemFoodData, ItemDrinkData, etc.)
ParseFuncFunction type: bool(const std::string&, std::vector<ItemType>&)
QualityCheckFuncFunction type: QualityChecker::QualityResult(const ItemType&)
WriteFuncFunction type: bool(const std::string&, const std::vector<ItemType>&)
GetExistingIdsFuncFunction type: std::set<std::string>(const std::string&)
BuildPromptFuncFunction type: std::string(...)
ProcessRecursiveFuncFunction type: int(...)
PostProcessFuncFunction type: void(std::vector<ItemType>&) - optional
Parameters
jsonResponseRaw JSON response from LLM
outputPathPath to output JSON file
requiredCountTarget number of items to generate
excludeIdsSet of existing item IDs to exclude (duplicates)
modelNameLLM model name for recursive calls
paramsGeneration parameters (maxHunger, maxThirst, etc.)
presetPreset type (Default, Forest, Desert, etc.)
itemTypeNameHuman-readable item type name (for logging)
registrySlugRegistry slug for ID tracking
parseFuncFunction to parse JSON into items
qualityCheckFuncFunction to perform quality checking
writeFuncFunction to write items to file
getExistingIdsFuncFunction to load existing IDs from file
buildPromptFuncFunction to build LLM prompt
processRecursiveFuncFunction to recursively generate more items if needed
[in,out]qualityResultsMap to store quality check results (used for filtering)
postProcessFuncOptional function to post-process items before validation
Returns
Number of items successfully added, or negative on error
Note
Saves raw JSON response to .raw.json file
Filters out duplicate IDs and invalid items (based on qualityResults)
Automatically merges with existing items in output file
Recursively generates more items if count is insufficient
